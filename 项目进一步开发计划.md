**【一、 项目当前已实现状态】**
目前我们的项目已经具备以下基础：
1. **泊松方程求解**：已实现静电场和热稳态场的求解，支持三类边界条件。
2. **线弹性应力场求解**：已实现求解及三类边界条件，包括使用罚函数方法处理的法向位移边界条件。
3. **单向耦合（电->热）**：已实现从电场到热场的单向耦合求解。
4. **单向耦合（热->力）**：已实现从热场到力场的单向耦合求解。
5. **并行处理**：已初步实现并行组装，但仍存留一些并行问题需要排查和解决。
6. **求解器架构**：已使用工厂模式（Factory Pattern）实现对不同线性求解器的选择，目前支持 amg、umfpack、pcg。

**【二、 本次核心开发目标】**
请逐步实现以下四个目标：
1. **优化并行组装**：排查并优化现有的并行组装流程，解决潜在的并行冲突或效率问题。
2. **稳态多物理场耦合**：实现电-热-力三向耦合求解。具体逻辑为：首先进行**电-热双向耦合求解**，在计算出最终的稳态温度场之后，再进行**热->力单向耦合求解**计算力场。
3. **瞬态多物理场耦合**：实现瞬态下的电-热-力三向耦合求解。具体逻辑同稳态：先进行电热双向耦合求解，最后算出稳态温度之后，再单向耦合求解力场。
4. **非线性处理（Picard迭代与muparser解析）**：
   - 针对目标2和目标3中的非线性（双向耦合）问题，统一使用 **Picard迭代法**。
   - 核心考虑材料属性受温度场影响的非线性变化。
   - 必须使用 **muparser** 库来解析定义在 JSON 文件中的材料属性函数。例如：电导率随温度变化的函数形如 `"1.0/(1.0+0.01*(T-300))"`，其中 `T` 代表当前的温度场变量。
